このシミュレータはぷよぷよクエストのぷよつかい大会におけるぬりかえをシミュレートし、素点が最大となるぬりかえを表示します.
1. 盤面空間の入力定義
 全て半角で入力. 以降の項目も同様.
 ・色情報: 1:赤, 2:青, 3:緑, 4:黄, 5:紫, 0:ハート で各セルを定義.
 ・プラス状態: プラスぷよを 1、通常状態を 0 と定義.
 ・ボーナスエリア: 対象セルを 1、非対象セルを 0 と定義.

2. ぬりかえ制約の設定
 ぬりかえ数を属性別に入力.
 ・塗 a: の最大ぬりかえ数
 ・塗 b: の最大ぬりかえ数

3. 演算時間の設定
 探索アルゴリズムの打ち切り時間(s)を入力. 単位は不要.
 （推奨値: 8s または 128s ）

4. 探索空間の定義
 探索深度 k ∈ ℕ0 を入力.
 (探索領域V = { (na, nb) | 塗a + 塗b - k <= na + nb <= 塗a + 塗b ; na <= 塗a, nb <= 塗b })

5. 演算の実行
 「探索開始」を押下. 計算処理がメインスレッドを占有するため、ブラウザから「応答なし」の警告が出る場合があるが、バックグラウンドでの収束計算は正常に進行中. 処理完了まで待機.

6. 解析結果の解釈と実行順序
 出力盤面上部に「なぞり消し座標」および「属性変換の優先順序」を表示.
 ・赤判定時: 赤からの変換が十分条件. 順序を誤るとぬりかえ時早期連結が発生し、なぞり消し前にオブジェクトが消滅する場合がある.
 ・青判定時: 青からの変換が必要条件. 順序を誤るとぬりかえ時早期連結が発生し、なぞり消し前にオブジェクトが消滅する.

🧪 開発目的
開発動機は主に3つあります
1つ目は、以前YouTubeにてぷよつかい大会でぬりかえスキルを使うときに「とにかく大量に塗り替え、潤沢ななぞり消し数に任せて1連鎖で大量に消す」というプレイスタイルを目撃したことです. 私としては、パズルゲームなのだから連鎖をしっかり繋げ,盤面のぷよ1つ1つを大切にしたいという思いがあります. そこで、このシミュレータで「連鎖を組むぬりかえ」の良さに気付き、いかに効率的で賢い塗り替え方を考えるかを重視してほしいと考えました. こうすることで、「強いキャラの枚数がランキングに直結する」という考えを薄め、盤面全体を上手く巻き込んだ「美しいぬりかえ方」を探求する楽しみを見出すことで、より深くゲームを楽しめるようになってほしいと思っています.
2つ目は、自分のぬりかえ能力がどれほどか知りたいという興味です. ぬりかえキャラを2体入れれば、ぬりかえパターンは8×10^19なので、普通のパソコンで頑張ってシミュレートしてもせいぜい数ppbしか探索できないため、ランダムにパソコンで探索するよりもまだ人が考えた方が点数が良くなるだろうということを検証するためにしています. 実際に検証してみたところ、自分が15分考えて188点に対し、プログラムだと10億パターンを60秒でやって174点という結果になり、やはり人間の思考の鋭さを再認識しました. (なお、本来はマルチスレッドによる高速化が可能ですが、自身のWeb実装に関する知見不足によりシングルスレッド動作に留まっており、計算速度は1割程度に低下しています.)
3つ目は、テスト期間など忙しい人がとりあえず「そこそこのスコア」を手軽にとれるようにし、ある程度のランキングに入れるようにすることで、ゲームがしたいという欲望を少しでも抑えられるようにするためです. 私自身、テスト期間中であっても「5,000位以内には入りたい」という思いから無理にプレイを続けてしまった経験があります。このツールを使うことで、同じような悩みを抱える人の状況を少しでも改善できればと考えています.
以上が私の開発動機です.
